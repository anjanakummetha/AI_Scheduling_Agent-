📱 iMessage Bridge (Mac Setup)

You can absolutely chat from your iPhone. The Mac acts as a background relay — it receives your iMessages and forwards them to Lexi. You send messages from your phone as normal; your Mac just needs to be on and running the bridge in the background.

How it works: Your iPhone sends an iMessage to the Apple ID signed into your Mac. The Mac’s OpenClaw gateway catches it and posts it to POST /webhooks/imessage on this server. Lexi responds, and OpenClaw delivers the reply back as an iMessage to your iPhone.

Grant Mac permissions
System Settings → Privacy & Security → Full Disk Access: add Terminal.
System Settings → Privacy & Security → Automation: allow Terminal to control Messages.
Install the imsg CLI on your Mac 
brew install steipete/formulae/imsg
Run the automated setup script
Copy the script at scripts/setup_imessage_mac.sh to your Mac, then: 
LEXI_SERVER_URL=https://your-server.com \
COMPOSIO_API_KEY=ak_ibLpD0PDrz458tUwldIh \
bash setup_imessage_mac.sh
Replace https://your-server.com with this server’s public URL. If running locally, use ngrok: ngrok http 8000 and use the generated URL.
Start the OpenClaw gateway 
cd ~/.lexi-openclaw && node gateway.js
Test it
From your iPhone, send an iMessage to the Apple ID signed into your Mac. Lexi will reply within a few seconds.
